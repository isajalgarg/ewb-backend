How this works
Step 1 — Sandbox JWT: Backend calls POST /authenticate with x-api-key + x-api-secret to get a Sandbox JWT.
Step 2 — EWB Token: Backend calls POST /gst/compliance/e-way-bill/tax-payer/authenticate with GSTIN + EWB credentials to get EWB portal token.
Step 3 — List EWBs: Backend calls GET /consignor/bills?generated_date={date} for each date in range.
Step 4 — Full Detail: For each new EWB, backend calls GET /consignor/bill/{ewbNo}. Existing EWBs preserve your ops data.
All API calls happen server-side. Your credentials never leave the server.
